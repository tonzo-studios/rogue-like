dungeon module¶

class dungeon.Dungeon(width, height)¶

Bases: object

Represents a dungeon / map in the game.

A dungeon is a place that the player explores, it is filled with enemies, items, npcs, secrets. It consists of a series of Room objects interconnected with narrow corridors.

After initialization, the dungeon is “unexplored”, i.e. the player can’t see the entire dungeon upon spawning. Once the player moves around and explores the dungeon, tiles will be revealed and will be “remembered”.

Parameters:
  • width (int) – Max width of the dungeon to be generated.
  • height (int) – Max height of the dungeon to be generated.
compute_fov(pos, fov, radius, light_walls)¶

Compute a FOV field from the passed-in position.

The FOV field represents how many tiles ahead can a certain actor see, this can affect behavior such as following, attacking, etc.

Parameters:
  • pos (Vector) – The position vector from which the FOV should be calculated, usually this position is occuppied by an Actor.
  • fov (str) – Type of FOV algorithm to be used, usually supplied by tdl itself.
  • radius (int) – How far away should the actor be able to see.
  • light_walls (bool) – Whether or not walls within the FOV of the actor should be lit up or not.

get_blocking_entity_at_location(pos)¶

Check if there’s a blocking entity at the specified location.

A blocking entity is one that turns the tile in which it resides into non-walkable, effectively blocking any other entities from accessing and or placing themselves into that same tile.

Parameters:pos (Vector) – Vector of the position to check for blocking entities.
Returns:The blocking entity if any, None otherwise.
